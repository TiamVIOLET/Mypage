# 图库模块设计方案

- 日期：2026-04-24
- 项目：MyPage
- 范围：首页图库预览、独立图库页、相册详情页、图库内容管理
- 方向：视觉展览感图库 + 系列相册管理

## 1. 背景与目标

当前站点已经有首页、角色档案、关于我、作品、音乐、文章、联系等页面，整体气质是蓝色夜景、二次元、列车与安静个人表达。用户希望新增一个图库模块，用来放自己收集的图片或生成的图片。

图库不只是图片存放区，而应该成为访客可浏览的视觉展览入口，同时也方便用户后续维护图片内容。

目标：

- 新增独立图库页，并在首页展示精选图库入口
- 用系列/相册组织图片，而不是普通散图列表
- 前台体验更偏“给访客逛”的视觉展览
- 支持三语界面
- 首版使用现有图片或渐变占位撑起效果
- 在 `editor.html` 提供完整图库管理能力

## 2. 已确认需求

- 入口形式：`B`，首页预览区 + 独立图库页
- 图片维护：`C`，先支持本地目录/配置，同时完整接入 editor 管理
- 图片组织：`C`，按系列/相册分组
- 系列浏览：`A`，进入独立详情页
- 用途定位：`D`，自己整理和访客浏览都要，但更偏访客浏览
- 多语言：`A`，中文 / 日文 / 英文全部支持
- 图库首页视觉：`C`，情绪大图 + 碎片拼贴
- 图片详情：`A`，只看大图，不展示额外信息
- 筛选检索：`B`，只保留系列入口，不做搜索和标签筛选
- 系列详情图片流：`B`，瀑布流
- 首批内容：`A`，用现有图片或渐变占位撑起效果
- editor 范围：`C`，完整相册/图片增删改管理

## 3. 范围与边界

### 3.1 本次范围

新增和改造：

- `gallery.html`：图库首页
- `gallery-detail.html`：动态相册详情页，通过 `?id=album-id` 渲染不同相册
- `index.html`：新增首页精选图库预览区
- 全站导航：新增图库入口
- `content.js`：新增图库默认数据、渲染逻辑、lightbox、editor 管理逻辑
- `style.css`：新增图库页面、瀑布流、lightbox、editor 图库管理样式
- `editor.html`：新增图库管理区域
- `assets/gallery/`：预留真实图库资源目录，首版可用占位或现有图片

### 3.2 暂不做

- 不做搜索
- 不做标签筛选
- 不做每张图片的标题、描述、时间等详情展示
- 不做服务端上传或云存储
- 不做多用户权限系统
- 不把每个相册拆成单独 HTML 文件

## 4. 信息架构

### 4.1 导航入口

主导航新增“图库”：

- 中文：图库
- 日文：ギャラリー
- 英文：Gallery

所有页面导航保持一致，当前处于图库相关页面时高亮图库入口。

### 4.2 首页精选图库

首页新增一个精选图库区块，用来提示站点新增了视觉收藏空间。

内容：

- 区块 badge：Gallery / visual collection 语义
- 标题与短说明，延续 Asa 的蓝色二次元语气
- 3 到 4 个精选图或精选相册入口
- 一个进入图库页的主按钮

目标：

- 不抢首页主视觉，但让访客能自然发现图库
- 与现有首页 quick card / section 风格兼容
- 后续可从图库数据中取 featured 内容

### 4.3 图库首页 `gallery.html`

图库首页采用“情绪大图 + 碎片拼贴”的视觉方向。

结构：

1. 顶部 hero：大幅情绪视觉，强调这是 Asa 的图片收藏/生成图展览空间
2. 精选拼贴：展示几张代表性图片或渐变占位，形成视觉冲击
3. 系列入口：相册卡片列表，每张卡片展示封面、标题、简短描述、图片数量、进入按钮

目标：

- 第一眼不像普通相册列表，而像视觉展览开场
- 系列入口足够清楚，访客知道下一步该点哪里
- 保留蓝色夜景、玻璃卡片、星光等现有站点语言

### 4.4 相册详情页 `gallery-detail.html`

通过 URL 参数选择相册：

- `gallery-detail.html?id=blue-night`
- `gallery-detail.html?id=generated-dreams`

结构：

1. 相册 hero：封面、标题、描述、图片数量、返回图库入口
2. 瀑布流图片区：按图片自然比例展示，偏图片站浏览体验
3. lightbox：点击图片后打开纯大图预览

无效 ID 或相册不存在时：

- 显示空状态
- 提供返回图库页入口

## 5. 数据设计

### 5.1 默认图库数据

在 `content.js` 的 `DEFAULT_CONTENT` 中新增 `gallery`。

建议结构：

```js
gallery: {
  albums: [
    {
      id: 'blue-night',
      title: '蓝色夜景收藏',
      description: '那些会让我停下来多看一眼的蓝色画面。',
      cover: './brand-avatar.jpg',
      featured: true,
      accent: '#84c7ff',
      mood: 'Blue night archive',
      images: [
        { src: './brand-avatar.jpg' },
        { src: '', placeholder: 'gradient-blue' }
      ]
    }
  ]
}
```

多语言内容可通过现有本地化机制覆盖。相册标题和描述需要支持中文、日文、英文。实现时可以采用与现有 `LOCALIZED_DEFAULT_CONTENT` 类似的结构，避免前台写死文案。

### 5.2 图片字段

图片首版只需要：

- `src`：图片路径或 data URL
- 可选 `placeholder`：当没有真实图片时使用渐变占位

不需要标题、描述、时间、标签等字段，因为用户选择了纯看图体验。

### 5.3 本地图片目录

预留目录：

- `assets/gallery/`

真实图片后续可以放入该目录，再在 editor 或默认数据里引用路径。

## 6. Editor 管理设计

`editor.html` 新增“图库管理”区域，进入 editor 解锁后可见。

能力：

- 新增相册
- 删除相册
- 切换当前相册
- 编辑相册标题、描述、封面、短标语、是否精选
- 新增图片
- 删除图片
- 编辑图片地址
- 上传图片文件并转为 data URL 存入浏览器本地内容
- 保存后写入 `localStorage`，前台刷新后可见

限制说明：

- 当前 editor 是纯前端本地管理，不会把文件真正写入仓库目录
- 上传图片转 data URL 适合少量图片预览；长期维护仍建议把图片放进 `assets/gallery/` 后填写路径
- 这个限制需要在 editor 帮助文案里说明清楚

## 7. 视觉与交互

### 7.1 视觉语言

延续现有站点：

- 深蓝渐变背景
- 玻璃卡片
- 柔和蓝光、星光、樱花/鼠标星星动效
- 二次元设定集式文案

图库专属方向：

- 更像视觉展览，不像资料表
- 大图和拼贴优先，文字克制
- 系列卡片带封面和数量，方便访客快速理解内容规模

### 7.2 瀑布流

相册详情图片区使用 CSS columns 或 grid masonry-like 布局。优先选择简单稳定的 CSS 方案，避免引入依赖。

移动端：

- 桌面端多列瀑布流
- 平板端两列
- 手机端单列

### 7.3 Lightbox

点击图片打开 lightbox：

- 只展示大图
- 提供关闭按钮
- 点击遮罩关闭
- `Esc` 关闭
- 不展示标题、描述、时间、标签

### 7.4 空状态

需要覆盖：

- 没有相册
- 相册无图片
- 图片路径为空时使用渐变占位
- `gallery-detail.html` 找不到指定相册

## 8. 多语言设计

需要补充以下三语文案：

- 导航图库入口
- 首页精选图库区块
- 图库首页 hero、按钮、空状态
- 相册详情返回按钮、空状态
- editor 图库管理区域标题、提示文案、按钮文案

相册数据也要支持三语默认内容。用户通过 editor 编辑时，至少应能编辑三语标题/描述，避免切换语言后内容缺失。

## 9. 测试与验收

实现完成后验证：

- 导航三语切换后，图库入口、页面标题、按钮文案正常变化
- 首页“精选图库”能显示默认内容并跳转到图库页或相册详情
- `gallery.html` 能展示情绪大图、精选拼贴和系列入口
- `gallery-detail.html?id=...` 能按相册 ID 渲染对应瀑布流
- 无效相册 ID 显示空状态和返回图库入口
- 点击图片打开纯大图 lightbox，遮罩、关闭按钮、Esc 都能关闭
- `editor.html` 能新增、编辑、删除相册和图片
- editor 保存后刷新前台可见
- 移动端布局不溢出

## 10. 实现原则

- 复用现有 `content.js` 本地内容管理模式，不引入后端
- 不添加不需要的搜索、标签筛选、图片元信息展示
- 尽量复用现有页面结构、导航结构和视觉组件
- 新增逻辑保持集中，避免散落到多个临时脚本
- 图片管理明确提示“本地浏览器保存”的限制
