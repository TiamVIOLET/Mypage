# 文章封面比例自适应展示设计

## 背景

当前文章封面直接放入固定高度容器，并使用统一的图片展示方式。不同封面比例差异较大时，横图、竖图和长图会出现留白突兀、主体过小或被硬裁切的问题，影响文章列表和详情页观感。

## 目标

让文章封面根据图片真实比例自动选择展示方式：横图保持沉浸感，竖图和长图完整保留主体，整体仍符合站点蓝色玻璃感与动漫氛围。

## 范围

本次只优化文章列表页和文章详情页的封面展示逻辑，包括：

- `articles.html` 动态渲染出的文章卡片封面。
- `article-detail.html` 动态渲染出的详情页封面。
- `content.js` 中文章封面渲染函数。
- `style.css` 中文章封面相关样式。

不修改 Markdown front matter、`assets/data/articles.json` 数据结构，也不要求用户手动维护图片比例。

## 推荐方案

采用比例自适应策略。渲染时仍由 `renderArticleCover(article, className)` 统一生成封面 DOM，但封面内部增加背景层和图片层：

```html
<div class="article-card-cover article-cover-frame is-loading">
  <img class="article-cover-backdrop" src="..." alt="" aria-hidden="true" />
  <img class="article-cover-image" src="..." alt="文章标题" loading="lazy" />
</div>
```

图片加载完成后，JavaScript 读取 `naturalWidth / naturalHeight`，给外层追加比例类：

- `is-landscape`：普通横图。
- `is-wide`：超宽横图。
- `is-square`：接近方图。
- `is-portrait`：竖图或长图。
- `is-error`：图片加载失败。

## 展示规则

### 横图

横图使用 `object-fit: cover` 铺满容器，保证文章卡片有完整的视觉冲击力。超宽图仍使用铺满策略，但通过更柔和的背景遮罩和稳定容器高度降低被硬裁切的感觉。

### 方图

方图使用 `object-fit: contain`，让图片完整显示，同时保持较大的主体尺寸。背景层使用同图模糊放大，避免固定容器内出现生硬空白。

### 竖图和长图

竖图、长图使用 `object-fit: contain` 完整展示。底层同图背景会被放大、模糊、降低透明度，形成柔和氛围填充，使留白更像设计的一部分，而不是图片没铺满。

### 加载失败

加载失败时保留深色渐变背景和边框，不显示破图图标，不影响文章标题、摘要和跳转。

## 数据流

1. 文章 Markdown 或 JSON 提供 `cover` 字段。
2. `content.js` 渲染封面容器、背景图和主体图。
3. 主体图加载完成后计算比例。
4. JS 将比例类写入封面外层。
5. CSS 根据比例类切换 `object-fit`、背景层显隐和视觉细节。

## 样式原则

- 文章列表封面高度保持稳定，避免网格跳动。
- 详情页封面高度更宽松，适合展示完整大图。
- 所有封面继续使用圆角、玻璃边框、暗色底和柔光遮罩，保持站点现有风格。
- 动效只使用轻微透明度和缩放变化，不引入复杂交互。

## 测试与验收

使用至少三类图片验证：

- 横图：铺满卡片，没有明显黑边或突兀留白。
- 竖图/长图：主体完整显示，不被裁掉。
- 方图：完整显示，背景填充自然。

同时确认：

- 文章列表页卡片高度稳定。
- 文章详情页封面展示自然。
- 图片加载失败时页面不破版。
- 语言切换和文章跳转不受影响。
