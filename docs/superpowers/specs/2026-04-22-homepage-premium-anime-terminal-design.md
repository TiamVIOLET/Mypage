# 首页重设计方案：Premium Anime Terminal

- 日期：2026-04-22
- 项目：MyPage
- 范围：仅首页 `index.html`
- 方向：**A1 / Premium Anime Terminal**

## 1. 背景与目标

当前首页已经有明确的蓝色夜景、列车、二次元氛围和角色表达，但整体信息组织仍接近常见的个人站 landing page：左侧 hero 文案、右侧场景展示、下方六宫格入口。这会削弱首页的独特辨识度，也容易产生“模板站 / 普通卡片官网”的观感。

本次重设计的目标不是换一套更流行的网页皮肤，而是把首页改造成一个**进入 Asa 世界后的角色主界面**：

- 保留蓝色夜景、列车路线、角色设定这三个核心记忆点
- 明显摆脱普通卡片官网感
- 避免做成冷冰冰的后台 dashboard
- 建立更强的“游戏 UI + 日系角色设定终端”识别度
- 先只重构首页，其他页面后续再统一

## 2. 用户确认后的设计结论

### 2.1 已确认方向

- 不要做成 aislop 风格
- 目标气质：**游戏 UI 感**
- 偏好子方向：**角色档案屏**
- 游戏化程度：**重游戏化**
- 希望保留：
  - 蓝色夜景
  - 列车路线
  - 角色设定
- 希望摆脱：
  - 普通卡片官网感
  - 后台 dashboard 感
- 改造范围：**先改首页**
- 视觉收敛版本：**A1 / Premium Anime Terminal**
- 进一步要求：**更二次元一点，同时更高级一点**

### 2.2 设计一句话总结

首页不再是“介绍 Asa 的 landing page”，而是“用户进入 Asa 世界后看到的角色终端主界面”。

## 3. 范围与边界

## 3.1 本次范围

只重构首页的前端表现层，重点涉及：

- `index.html:18-50` 顶部导航
- `index.html:54-117` 首页 hero 区
- `index.html:119-165` 首页入口区
- `style.css:522-905` 一段首页主样式
- `style.css:393-465` 现有通用卡片视觉语言在首页的应用方式
- `script.js:76-96` 与首页卡片 hover / reveal 相关的交互表现

## 3.2 暂不变更

以下内容本次不改：

- `content.js` 的内容数据结构
- 多语言机制
- 多页面跳转关系
- Live2D 功能逻辑
- 音乐功能逻辑
- 其他内页的结构与视觉体系

## 4. 首页信息架构

## 4.1 顶部 Terminal Bar

保留现有多语言与多页面能力，但把视觉语义从“胶囊站点导航”改为“终端控制栏”。

### 结构

- 左侧：站点标识 / terminal id / project label
- 中部：带编号的主导航
  - `01 HOME`
  - `02 PROFILE`
  - `03 ABOUT`
  - `04 WORKS`
  - `05 LOGS`
  - `06 MUSIC`
- 右侧：状态标签，如 `ONLINE / ACTIVE`
- 保留语言切换区

### 目标

- 功能不变
- 视觉语义变成系统入口栏
- 降低“个人主页导航条”的既视感

## 4.2 左侧主角色面板

左侧成为首页第一视觉焦点，用来承载角色身份与主操作。

### 内容构成

- 角色名：`ASA`
- 角色属性：type / build / mood / traits
- 角色 visual slot（预留角色视觉位）
- 一段简洁的角色状态说明文案
- 主按钮：
  - `OPEN PROFILE`
  - `VIEW WORKS`

### 目标

- 用户第一眼先认识“角色”，不是先读一段网页介绍
- 文案从自我介绍转为角色状态说明
- 建立强烈的“角色档案屏”认知

## 4.3 右侧路线导航主视觉

右侧不再是偏插画用途的场景展示，而是“蓝色线路图 + 站点高亮”的可操作导航主视觉。

### 内容构成

- Blue Line 路线主线
- 当前站点高亮
- Scene label / route code / active stop
- 站点与页面入口对应关系

### 目标

- 保留列车路线意象
- 把夜景和路线从“装饰”变成“导航逻辑的一部分”
- 让首页更像主界面，而不是情绪插画卡

## 4.4 底部辅助模块

现有六宫格 `quick-grid` 是最接近模板站的部分，需要彻底重做为 sub terminal deck。

### 新分层

- 主入口：Profile / Works / Logs
- 辅助入口：Music / Contact / About

### 表现方式

- 编号式入口模块
- 子系统面板
- 更少的卡片感，更强的面板感

### 目标

- 去掉平铺官网入口感
- 建立首页的信息主次
- 让入口区像“子终端模块”，不是营销卡片墙

## 4.5 Footer

页脚保留信息收束能力，但视觉语义改成 terminal footer / project credit / system end note。

### 目标

- 弱化普通网页收尾文案感
- 保持从页首到页尾的一致界面语言

## 5. 视觉语言规范

## 5.1 核心定位

视觉方向确定为：**Premium Anime Terminal**。

这不是：

- 毛玻璃悬浮卡片站
- SaaS 仪表盘
- 夸张 neon 科幻 UI
- 可爱装饰堆砌的二次元模板

这是：

- 日系角色设定终端
- 带夜色与站线意象的高级游戏 UI
- 更像角色设定集 + 主界面，而不是普通官网

## 5.2 配色系统

首页配色使用 5 个层级：

- `Night Black`：深夜底色，提供高级感和纵深
- `Deep Panel Blue`：主面板底色
- `Signal Blue`：系统高亮色
- `Route Mint`：路线 / 状态条高亮
- `Station Amber`：少量暖光点缀，模拟站台灯或角色高光

### 规则

- 深色占主导
- 高亮只用于关键边界、状态条、当前站点
- 暖色只做点缀，不扩大面积
- 不做全局发光污染

## 5.3 面板质感

首页将摆脱现有通用玻璃卡片语言，改为更硬朗、更精致的 panel 系统。

### 规则

- 使用细边框、内嵌分隔线、顶部扫光线
- 减弱大面积 `backdrop-filter` 玻璃感
- 模块角落增加 sector label / system code
- hover 时以边框激活、局部亮线为主
- 弱化浮起感与柔软胶囊感

### 目标

让页面像“角色终端面板”，而不是“常见网页卡片”。

## 5.4 二次元感来源

二次元感不通过幼态装饰实现，而通过以下方式建立：

- 角色设定集式信息组织
- 角色名、type、build、mood、traits 的资料语言
- 角色 visual slot
- scene label / route code / active stop 等界面语义
- 文案语气更像“角色状态说明”

### 目标

让首页更像高质量日系角色主页，而不是套模板的二次元站。

## 5.5 高级感来源

高级感来自克制，而不是堆满效果。

### 规则

- 高光少而准确
- 颜色点亮有节制
- 留白和层级整齐
- 编号、标签、标题、正文有统一节奏
- 不做夸张 neon、故障风、赛博噪点堆叠

### 目标

看起来像有审美控制力的游戏 UI，而不是过度设计的概念稿。

## 6. 首页组件落地策略

## 6.1 头部导航

现有 `site-header / nav-wrap` 保留结构能力，但改造成 terminal nav bar。

### 具体策略

- `brand` 从普通站点 logo 语义改为 terminal id
- 主导航增加编号前缀
- 导航条整体更扁、更硬朗、更像控制栏
- 语言切换保留，但样式融入系统条

## 6.2 主 Hero 区

保留双栏框架，但彻底改变内容语义。

### 左侧 `hero-copy`

- 去掉普通 hero 文案逻辑
- 改成角色档案主面板
- 现有 `hero-points` 改造成状态参数 / system info 小模块

### 右侧 `hero-visual`

- 现有 `scene-card` 从情绪插画卡转为路线导航面板
- 保留夜景、站线、列车意象
- 强化“可操作导航主视觉”定位

## 6.3 入口区

现有 `quick-grid` 六张卡片重构为 sub terminal deck。

### 原则

- 不再是平铺 landing page 卡片
- 改为主次分层的子系统模块
- 主入口优先级高于辅助入口
- 保留所有原有内容跳转，不改变信息去向

## 6.4 页脚

- 保留内容收束和版权信息
- 改为终端尾注风格
- 弱化“感谢路过我的小宇宙”这类普通官网结尾语义

## 7. 文案策略

首页文案从“个人介绍型 hero”改成“角色状态说明 + 系统引导语”。

### 替换方向

从：

- 我是 Asa……
- 如果你想更快认识我……

转为：

- `PLAYER DOSSIER`
- `CURRENT STATUS`
- `OPEN PROFILE`
- `ROUTE NAVIGATION`
- `SYSTEM NOTE`

### 原则

- 保留原有安静、蓝色、慢热、番剧感内核
- 改变表达方式，不改变人物气质
- 文案更短、更准、更像界面的一部分

## 8. 动效与交互

## 8.1 要保留的感觉

- 轻微终端唤醒感
- 路线在线感
- 状态条推进感
- 柔和但利落的 hover 反馈

## 8.2 要收敛的内容

现有首页以下交互会被收敛或重写：

- `script.js:76-85` 的强烈鼠标跟随高光
- `script.js:57-74` 的鼠标星星装饰效果
- 基于卡片上浮的 hover 主反馈

## 8.3 新交互规则

- 卡片不再以“上浮”作为主要反馈
- 面板 hover 以边框激活、亮线、局部高亮为主
- 当前路线站点做轻微呼吸或脉冲
- 按钮更像操作指令：`OPEN / VIEW / ENTER`
- reveal 动画更偏横向切入或系统激活，不再是网页式悬浮显现

## 9. 响应式策略

移动端仍保持可用，不因为游戏 UI 化而牺牲可读性。

### 规则

- 首页改为纵向堆叠
- 顺序固定为：
  1. 顶部 terminal bar
  2. 左侧主角色面板
  3. 右侧路线导航面板
  4. 底部辅助模块
- 保持主次明确
- 保证按钮可点、文案可读、导航不拥挤

## 10. 验证标准

以下 5 条同时满足，视为首页重设计成功：

1. **首屏一眼看上去就是角色主界面**
   - 不是普通 landing page
   - 不是后台 dashboard

2. **人物面板是第一焦点**
   - 路线导航是第二焦点
   - 信息层级清晰

3. **二次元感来自设定语言和界面节奏**
   - 不是依赖装饰堆砌

4. **高级感来自克制**
   - 高光少而准
   - 边框、留白、标签、层级更精确

5. **移动端仍然成立**
   - 不因视觉风格变化而变得难用

## 11. 实施影响文件

本次首页重设计预期主要影响以下文件：

- `index.html`
- `style.css`
- `script.js`

不预期改动：

- `content.js`
- 其他内页 html

## 12. 非目标

以下不属于本次首页方案范围：

- 整站统一视觉系统一次性改完
- 新增后台功能或内容管理能力
- 改造现有内容模型
- 引入框架、构建工具或新的设计系统依赖
- 添加与首页无关的新特效

## 13. 最终结论

首页最终采用 **A1 / Premium Anime Terminal** 方案。

这是一个以“角色档案终端”为核心、以“蓝色路线导航”为第二焦点、同时兼顾二次元气质与高级感的首页重设计方向。它会明显区别于常见模板站，也不会滑向冷硬后台，而是建立一个更像 Asa 本人、也更有记忆点的首页世界观。
