# 后端开发定位词汇替换设计

## 背景
当前个人站整体是蓝色动漫风格，用户希望去掉展示内容中明确的前端开发定位，并统一替换为后端开发方向。用户后续明确收窄范围：不能改动漫、蓝色、夜景、BGM、情绪氛围等文案，只做明显词汇替换。

## 范围
- 仅把展示文案中的 `前端`、`Frontend`、`フロントエンド` 替换为 `后端`、`Backend`、`バックエンド`。
- 覆盖 `content.js`、`script.js`、`profile.html` 等站点展示来源中的明显前端定位词汇。
- 保留网站现有页面结构、导航、蓝色动漫氛围、图库、音乐、编辑器与本地内容管理功能。
- 不修改用户上传或导入的文章内容，包括 `content/` 下的文章源文件和 `assets/data/articles.json` 中已生成的文章数据。

## 内容边界
- 不整体重写作品、文章、关于、联系、音乐等情绪文案。
- 不把“页面、动效、视觉、BGM、夜景、分镜”等非明确前端词汇改成后端系统叙述。
- 默认示例文章中若出现明确的前端词汇，只做同等粒度替换，不改文章结构和情绪表达。

## 验证
- 对站点展示文件搜索 `前端`、`Frontend`、`フロントエンド`，确认除测试断言外不再出现。
- 运行新增的后端定位回归测试。
- 确认 `content/` 与 `assets/data/articles.json` 未被修改。
