## 一、プロジェクト概要

健康的な食生活、科学的な栄養管理、そして生活の質に対する関心が高まるにつれ、経験に頼って食事を組み立てる従来の方法だけでは、パーソナライズ化、利便性、知能化へのニーズを十分に満たすことが難しくなっています。ユーザーごとに身体目標、食習慣、アレルギー、手元にある食材、栄養摂取状況は大きく異なります。そのため、ユーザーの実情を踏まえて知的に分析し、提案できる食事支援システムには高い実用的意義があります。

**AI Chef Mate** は、このような背景のもとで設計・実装された、人工知能によるパーソナライズ食事・栄養システムです。本プロジェクトは「食材認識、スマート献立生成、献立管理、栄養トラッキング」という四つの方向を中心に、画像認識、自然言語生成、データ記録、Web インタラクション体験を組み合わせ、比較的完整なスマート食事管理フローを形成しています。ユーザーは食材の画像をアップロードするか、食材を手動で入力できます。システムは食の好み、アレルギー情報、タンパク質摂取目標を組み合わせて、パーソナライズされた献立を生成します。また、ユーザーは過去の献立を保存し、対話を通じて提案をさらに最適化し、栄養トラッキング機能で毎日の食事状況を記録することもできます。

全体として、本プロジェクトは単なる機能デモにとどまらず、日常的な健康管理シーンにおける人工知能技術の実用化可能性を示しています。これまで分散していた「食材を見る、レシピを考える、栄養を計算する、記録する」といったステップを一つの流れに統合することで、食事計画をより直感的で効率的、かつ知的なものにしています。

## 二、システムの主要機能

### 1. 食材認識と入力

システムは、画像アップロードによる食材認識に加え、ユーザーによる食材情報の手動追加、修正、削除にも対応しています。この設計は知能化と柔軟性の両方を考慮したものです。一方では、画像認識によってユーザーの入力負担を減らし、システムを実生活の利用場面に近づけます。もう一方では、手動入力機能によって、ユーザーが実際の状況に応じて認識結果を補足できるため、モデル出力への完全な依存を避けられます。

利用フローでは、ユーザーはまず冷蔵庫、キッチン、または食材の写真をアップロードし、システムが画像内容に基づいて利用可能な材料を整理します。その後、ユーザーは不足している食材を追加し、より正確な食材リストを作成できます。この機能は後続の献立生成に必要な基礎データを提供し、システム全体に初期段階から高い実用性を持たせています。

### 2. パーソナライズされた献立生成

献立生成は、本プロジェクトにおける最も中心的な機能の一つです。システムは、ユーザーが提供した食材、食の好み、アレルギー情報、毎日のタンパク質目標に基づき、構造の明確な一週間分の献立を生成します。一般的なレシピ検索と比べて、本プロジェクトは「パーソナライズ」と「調整可能性」をより重視しています。ユーザーは固定された結果を受け取るだけではなく、自分の条件に合わせて、よりニーズに合った提案を生成できます。

生成された献立は、対話によってさらに最適化できます。たとえば、特定の食材の置き換え、脂質の削減、タンパク質比率の向上、作り置き方法の調整、味の方向性の変更などを依頼できます。この対話型インタラクションにより、献立は一度きりの結果ではなく、継続的に修正できる知的な提案になります。

### 3. 献立の保存と管理

システムは保存済み献立の管理機能を提供します。ユーザーは過去の献立を確認し、献立名を変更し、不要な記録を削除し、ポップアップで完全な Markdown 形式の献立内容を閲覧できます。この機能により、システムは即時生成だけでなく、継続的に利用できる価値を備えています。

ユーザーにとって、献立を保存することは、異なる時期の食事計画を比較したり、自分に合った提案を後から再利用したりする助けになります。プロジェクトの展示という観点からも、このモジュールはデータ管理とユーザー体験の両面における完成度を示しています。

### 4. 栄養トラッキングと集計

献立生成に加えて、システムは栄養トラッキング機能も提供しています。ユーザーは食べ物の写真をアップロードするか、食品名を入力することで、システムがカロリー、タンパク質、炭水化物、脂質などの主要な栄養指標を分析・記録します。ユーザーは分析結果を手動で修正することもでき、最終的な記録を実際の状況により近づけられます。

システムは朝食、昼食、夕食、間食などの食事区分に基づいて日々の食事記録を整理し、過去一週間の栄養集計も提供します。このモジュールによって、ユーザーは「食事を計画する」段階から「食事を記録する」「食事を振り返る」段階へと進むことができ、健康的な食生活の完整なループを形成できます。

## 三、技術実装のポイント

### 1. Next.js に基づくフルスタックアプリケーション構成

本プロジェクトの現在の主要実装は、Next.js のフルスタック構成を採用しています。フロントエンド画面、サーバー API、データの読み書き、ビジネスロジックはいずれも `src/` ディレクトリ配下で実装されています。前後端を完全に分離する従来型の構成と比べて、この方式は卒業制作の展示やローカルデモにより適しています。デプロイの流れが簡潔で、画面と API の連携も直接的であり、開発効率も高くなります。

システムは Next.js App Router を用いてページと API を整理しており、主なページにはトップページ、ダッシュボード、献立作成、献立チャット、保存済み献立、栄養トラッキングなどがあります。サーバー API は、チャット生成、食材認識、栄養分析、献立保存、献立照会、栄養記録などをカバーしており、プロジェクト全体に比較的完整な業務ループを持たせています。

### 2. モダンなフロントエンド技術スタック

フロントエンド部分は、React、TypeScript、Tailwind CSS、shadcn/ui などの技術で構築されています。React はコンポーネント指向の開発体験を支え、TypeScript はコードの型安全性と保守性を高めます。Tailwind CSS は画面スタイルの開発を効率化し、shadcn/ui は比較的統一されたコンポーネント基盤を提供します。

画面表現の面では、システムはダーク背景、グラスモーフィズム風のカード、グラデーションボタン、明確なモジュール型レイアウトを採用しており、全体として現代的な Web アプリケーションらしい質感を持っています。トップページではシステムの価値を直感的に示し、ダッシュボードでは機能入口を明確に分け、献立作成フローではステップ形式のガイドを採用し、栄養トラッキングページではデータカードを通じて日次・週次の栄養状況を表示します。これらの設計は、システムの使いやすさと展示効果を高めています。

### 3. AI 機能の柔軟な接続

本プロジェクトは、AI 機能の接続において柔軟な設計を採用しています。デフォルトでは、ローカルのデモロジックを使って食材認識、献立生成、栄養推定を行えるため、外部 API Key がない場合でもプロジェクトを実行・展示できます。一方で、智譜 AI API Key が設定されている場合には、実際のモデルを優先的に呼び出し、テキスト生成と視覚認識の能力をさらに強化できます。

この「二つのモード」の設計には明確な利点があります。一方では、プロジェクトの実行ハードルを下げ、発表、授業展示、ローカルデバッグを容易にします。もう一方では、将来的により強力なモデル能力を接続する余地を残しており、システムがデモプロトタイプから実用アプリケーションへ発展する可能性を持たせています。

### 4. シンプルさと拡張性を両立したデータ保存方式

システムはデフォルトでローカル JSON ファイルをデータ保存に使用し、献立、栄養記録、チャット状態を保存します。この方式はシンプルで直感的であり、ローカル開発やデモに適しています。同時に、プロジェクトは MySQL ストレージにも対応しており、データベース環境変数を設定すればリレーショナルデータベースによる保存へ切り替えることができます。

この設計は、使いやすさと拡張性を両立しています。卒業制作の段階では、ローカル保存によって環境構築の難度を下げられます。将来的なプロダクト化や複数ユーザー利用の場面では、MySQL によってより安定したデータ管理、検索、拡張を実現できます。

### 5. 初期研究実装を残し、プロジェクトの発展過程を示す

リポジトリには、初期段階の FastAPI、CrewAI、Qdrant、および関連する実験資料も残されています。これらは現在のフロントエンドのデフォルト依存となる主要経路ではありませんが、プロジェクトが Agentic RAG、知的エージェント協調から、現在の実行可能な Web デモシステムへと探索してきた過程を示しています。

この点は卒業制作プロジェクトにおいて特に価値があります。最終成果だけでなく技術ルート探索の痕跡も示すことで、実装過程におけるアーキテクチャ調整、技術選択、機能の焦点化を説明できるからです。

## 四、プロジェクトの特徴と強み

### 1. 完整な機能ループ

AI Chef Mate の最大の特徴は、単一のレシピ生成ツールではなく、スマート食事管理を中心に比較的完整なループを構築している点です。ユーザーはまず食材を入力または認識し、次にパーソナライズされた献立を生成し、その後献立を保存・調整し、最後に栄養トラッキングを通じて実際の摂取状況を記録します。この流れは「計画」から「実行」、そして「振り返り」までの主要な段階をカバーしています。

### 2. 高いパーソナライズ性

システムは献立生成時に、食材の在庫、食の好み、アレルギー情報、タンパク質目標など、複数のユーザー要素を考慮します。キーワードだけでレシピを検索する方法と比べて、この提案は実際のユーザーニーズにより近く、人工知能による意思決定支援の価値をより明確に示します。

### 3. 使いやすいユーザー体験

プロジェクトは、ダッシュボード、ステップ形式のフォーム、ポップアップ、カード型レイアウト、提示メッセージ、Markdown 献立表示などを通じて、ユーザーの理解と操作にかかる負担を軽減しています。技術的な背景を持たないユーザーでも、画面の案内に従って食材入力から献立生成までの流れを完了できます。

### 4. 高いデモ安定性

システムはデフォルトでローカルデモ AI とローカル JSON 保存を提供しているため、外部モデルサービスやデータベース環境がなくても主要なフローをデモできます。これは卒業制作の発表、プロジェクト報告、ローカルテストにとって非常に重要であり、ネットワーク、API Key、データベース設定に起因する展示リスクを減らせます。

### 5. 十分な将来拡張性

プロジェクトはすでに、フロントエンドページ、業務コンポーネント、API ルート、サーバー側 AI 呼び出し、ローカル保存、MySQL 保存など、比較的明確なモジュール分割を備えています。この構造は今後の継続的な改善の基盤となり、モデル能力、ユーザーシステム、データ可視化、モバイル対応、推薦アルゴリズムなどの方向へ拡張できます。

## 五、応用価値

実際の応用という観点から見ると、このシステムは複数の場面で利用できます。

一般ユーザーにとっては、手元の食材に基づいて食事を素早く計画し、「今日は何を食べるか」という意思決定コストを下げられます。また、栄養トラッキングを通じてカロリーや栄養構成への意識を高めることもできます。

フィットネスや健康管理を行う人々にとっては、システム内のタンパク質目標、三大栄養素の記録、週次集計が一定の参考価値を持ち、より意識的な食事管理を支援できます。

学生プロジェクトや技術展示としては、このシステムはフロントエンド開発、バックエンド API、データ保存、画像認識、自然言語生成、ユーザーインタラクション設計を組み合わせており、人工知能アプリケーション開発能力を比較的総合的に示せます。

将来的なプロダクト化の探索においても、本プロジェクトはスマート食事アシスタント、健康管理プラットフォーム、家庭向け食材管理システム、栄養相談ツールのプロトタイプ基盤として活用できます。

## 六、今後の拡張方向

### 1. より強力なマルチモーダル AI モデルの接続

今後は、より高度なマルチモーダルモデルを接続し、食材認識、料理認識、栄養推定の精度を高めることができます。システムは、複数の食材が混在する画像の認識、食物重量の推定、皿全体の構成分析など、より複雑な画像シーンにも対応できるようになり、栄養計算結果を実際の状況により近づけられます。

### 2. より充実したユーザープロファイルの構築

現在のシステムは、食の好み、アレルギー情報、タンパク質目標に対応しています。今後は、年齢、性別、身長、体重、運動頻度、健康目標、疾患に伴う禁忌、味の好み、予算範囲などのユーザープロファイルをさらに拡張できます。これにより、生成される献立はより個別化され、長期的な利用価値も高まります。

### 3. 専門的な栄養知識ベースの導入

今後は、権威ある食品栄養データベースや食事ガイドラインを接続し、AI 生成結果と構造化された栄養知識を組み合わせることができます。これにより、献立提案の科学性を高めると同時に、純粋なテキストモデル生成で起こり得る不正確な内容を減らせます。

### 4. 推薦アルゴリズムと長期計画機能の最適化

システムは今後、ユーザーの履歴記録から好みを学習し、個人の習慣により合った献立を徐々に推薦できるようになります。たとえば、ユーザーが高タンパク食をよく選ぶ、薄味を好む、特定の食材を好まないといった傾向を、後続の生成時に自動で反映できます。また、減脂、筋力増加、血糖コントロール、バランス食などの長期目標計画にも対応できます。

### 5. データ可視化機能の強化

栄養トラッキングモジュールには、折れ線グラフ、棒グラフ、円グラフ、傾向分析などを追加できます。これにより、ユーザーはカロリー、タンパク質、炭水化物、脂質の変化をより直感的に観察できます。システムは週報、月報、目標達成率も提供でき、食事管理により強いフィードバック感を与えられます。

### 6. アカウント体系と権限管理の整備

現在の認証ロジックはデモ用途に近いものです。今後は、Supabase Auth、NextAuth/Auth.js、または独自の認証システムなど、より完整なユーザー認証方式を導入できます。これにより、登録、ログイン、パスワードリセット、権限管理、複数デバイス同期を実現し、実利用シーンの基盤を提供できます。

### 7. モバイル端末とクロスプラットフォーム体験の拡張

食事記録はスマートフォンで行われることが多いため、今後はモバイル対応をさらに最適化し、ミニアプリやモバイルアプリの開発も検討できます。ユーザーは料理中、買い物中、食事中にいつでも写真で記録でき、システムを日常生活により自然に組み込めます。

### 8. スマートハードウェアや外部サービスとの連携

将来的には、スマート体組成計、フィットネストラッカー、健康プラットフォーム、買い物リスト、フードデリバリーサービスなどと連携することもできます。たとえば、運動消費量に応じて献立を調整したり、冷蔵庫の在庫に基づいて料理を推薦したり、買い物リストで不足食材を補ったりできます。これらの方向性は、システムの知能化と実用性をさらに高めます。

### 9. 本番環境レベルの安定性向上

プロジェクトをさらにプロダクト化していく場合、API セキュリティ、データ検証、ログ記録、エラー監視、データベースバックアップ、性能最適化、自動テストなどを継続的に整備できます。これにより、システムを卒業制作のデモ版から、より安定した実用アプリケーションへ段階的に発展させられます。

## 七、まとめ

総じて、AI Chef Mate はテーマが明確で、フローが完整であり、展示効果にも優れた人工知能による食事・栄養管理システムです。食材認識、パーソナライズ献立生成、対話型調整、献立保存、栄養トラッキングを一つの Web アプリケーションに統合し、健康的な食生活分野における人工知能技術の応用価値を示しています。

本プロジェクトの強みは、機能数だけでなく、全体フローの一貫性と拡張可能性にもあります。卒業制作プロジェクトとして完整に展示できるだけでなく、スマート食事アシスタントへ継続的に発展させる基盤も備えています。マルチモーダル AI、パーソナライズ推薦、栄養知識ベース、モバイル体験がさらに整備されていけば、このシステムは将来的に個人の健康管理、家庭の食事計画、スマート栄養サービスなどの場面で、より大きな役割を果たす可能性があります。
